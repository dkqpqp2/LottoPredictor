# DB Migrations

이 디렉토리의 `.sql` 파일들은 Supabase Postgres에 **수동으로** 적용해야 합니다. 백엔드는 `spring.jpa.hibernate.ddl-auto=validate`로 설정되어 있어서, 엔티티와 실제 DB 스키마가 일치하는지 검증만 하고 스스로 테이블을 만들거나 바꾸지 않습니다.

**중요:** 새 마이그레이션이 추가된 백엔드 버전을 배포하기 **전에** 반드시 해당 SQL 파일을 Supabase에 먼저 적용하세요. 순서를 바꾸면 (마이그레이션 미적용 상태로 먼저 배포하면) Hibernate 스키마 검증 실패로 백엔드 전체가 시작되지 않습니다 — 새로 추가된 기능뿐 아니라 기존 기능까지 전부 영향을 받습니다.

파일은 `000N_설명.sql` 형식으로 순번을 매겨 추가합니다.
