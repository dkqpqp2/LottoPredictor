# 소개/랜딩 페이지 추가 설계

## 배경 및 목적

현재 홈(`/`)은 타로 페이지다. 사이트에 처음 들어온 유저가 무슨 서비스인지 바로 파악하기 어렵다는 문제가 있어, 서비스 소개용 랜딩 페이지를 홈에 두고 타로 페이지는 별도 경로로 옮긴다. (참고: Soccer Position Management 랜딩 페이지의 히어로 + CTA + 기능 카드 그리드 구조에서 레이아웃 아이디어만 참고, 문구/브랜딩은 로타로 고유의 것으로 작성)

## 범위

**포함:**
- 새 소개/랜딩 페이지 (`/`)
- 기존 타로 페이지를 `/tarot`로 이동
- Nav에 "타로" 메뉴 추가
- sitemap에 `/tarot` 추가

**제외:**
- 기존 타로/통계/생성/회차조회 페이지 자체의 기능 변경 없음 (파일 이동만)
- 등급제/포인트 시스템 (다음 스펙)

## 라우팅 변경

- `frontend/app/page.tsx` (타로 페이지, 현재) → `frontend/app/tarot/page.tsx`로 이동
  - `frontend/app/page.module.css` → `frontend/app/tarot/page.module.css`로 이동
  - 파일 내 상대경로 import를 한 단계 더 깊게 조정 (`../lib/...` → `../../lib/...`, `../contexts/...` → `../../contexts/...`)
- `frontend/app/page.tsx` (신규): 소개/랜딩 페이지가 이 경로를 차지
- `frontend/app/components/Nav.tsx`: `LINKS` 배열에 `{ href: "/tarot", label: "타로" }` 추가 (홈 다음, 번호생성 앞)
- `frontend/app/sitemap.ts`: `ROUTES` 배열에 `{ path: "/tarot", priority: 0.9 }` 추가

## 소개 페이지 콘텐츠

- **히어로**
  - 타이틀: "로타로"
  - 서브타이틀: "타로 카드와 로또 통계로 만나는, 나만의 특별한 번호"
- **메인 CTA**: "타로 보러가기" 버튼 → `/tarot`
- **기능 카드 3개** (그리드, 각 카드 클릭 시 해당 경로로 이동):
  1. **타로 운세** — "카드를 뽑고 방향을 정해 나만의 타로 리딩과 행운의 번호를 받아보세요." → `/tarot`
  2. **번호 생성** — "역대 당첨번호 출현 빈도를 반영한 가중치 방식으로 번호를 뽑아드려요." → `/generate`
  3. **통계 & 회차조회** — "번호별 출현 통계와 역대 당첨번호를 한눈에 확인하세요." → `/stats`

## 디자인 톤

타로 페이지(`/tarot`)의 우주/신비 다크 콘셉을 그대로 재사용한다 — `--cosmic-bg`, `--cosmic-surface`, `--cosmic-border`, `--cosmic-text`, `--cosmic-gold` 등 기존 CSS 커스텀 프로퍼티와 별이 흩뿌려진 배경 그라디언트를 새 랜딩 페이지의 CSS 모듈에도 동일하게 적용한다 (사이트 라이트/다크 모드와 무관하게 항상 다크, 타로 페이지와 동일한 정책). 사이트 전체적으로 처음 들어왔을 때부터 타로 페이지로 넘어갈 때까지 시각적 일관성을 준다.

## 데이터 흐름 & 에러 처리

정적 콘텐츠 페이지라 API 호출이 없다. 로그인 상태 표시는 기존 `Nav.tsx`가 그대로 처리하며 이 페이지는 관여하지 않는다.

## 테스트

React 컴포넌트라 기존 컨벤션(Global Constraints 참고: `environment: 'node'`, jsdom 없음)대로 자동 테스트는 작성하지 않는다. `npx tsc --noEmit` 통과 + 브라우저 수동 확인(홈 접속 시 랜딩 페이지 표시, 카드 3개 클릭 시 각각 올바른 경로 이동, `/tarot` 접속 시 타로 페이지 정상 렌더링, Nav의 "타로" 링크 활성 상태 표시)으로 검증한다.
